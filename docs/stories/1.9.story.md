# Story 1.9: WhatsApp Web Simulator Foundation

**Epic**: WhatsApp Web Simulator Foundation  
**Story ID**: 1.9  
**Priority**: High  
**Estimated Effort**: 8 story points  
**Status**: Ready for Review

## Story

**As a** a developer like mw who will be testing each and everfeature of salex,  
**I want** a WhatsApp Web simulator that allows me to interact with my business through a familiar WhatsApp-like interface,  
**so that** I can test complete booking flows using the same backend logic as real WhatsApp integration without external API dependencies.

## Acceptance Criteria

### AC1: Webhook-Based Architecture
- Mock UI sends WhatsApp Cloud API compatible payloads to existing `/webhooks/whatsapp` endpoint
- All message processing, business routing, and conversation flow handled by existing webhook logic
- Environment flag `WHATSAPP_MODE=simulator` distinguishes between mock and real WhatsApp responses
- No changes to existing `/apps/api/src/modules/whatsapp/` production code

### AC2: Simple HTML/CSS/JS Interface
- Single chat interface in `/WhatsappMockUI/` directory
- WhatsApp-style chat with verified business account "Salex"
- No contacts list, no extra features - just one open chat conversation
- Mobile-responsive design matching WhatsApp Web aesthetics

### AC3: Business Code Integration
- Users can enter routing codes (S1234) to connect with real businesses from database
- Integration with completed 4-digit business identifier system (Story 1.8)
- Business discovery through routing code parsing

### AC4: Message Flow Implementation
- Customer messages → Webhook → Business logic → Automated responses → UI polling
- Response polling mechanism to retrieve and display business responses
- Sub-3 second message delivery with efficient caching

### AC5: Database Schema Extensions
- Customer, Conversation, and Message tables with foreign key relationships to existing Business model
- PostgreSQL JSONB for message content storage supporting rich WhatsApp message types
- Maintain existing booking workflow with WhatsApp conversation context

## Dev Notes

### Prerequisites
- Story 1.8 (4-Digit Business Identifier System) must be completed
- Existing `/webhooks/whatsapp` endpoint must be functional
- Business routing codes (1234 format) must be available in database

### Technical Implementation Strategy
1. **Isolated Simulator Module**: All code in `/apps/api/src/modules/whatsapp-simulator/`
2. **Webhook Enhancement**: Extend existing webhook with simulator response storage
3. **Simple Frontend**: HTML/CSS/JS without complex frameworks
4. **Environment-Based Switching**: Production-ready migration path

### File Structure
```
apps/api/src/modules/whatsapp-simulator/
├── mock-response.service.ts     # Store/retrieve responses for UI polling
├── webhook-enhancer.service.ts  # Extend webhook with simulator support
├── simulator.controller.ts      # Polling endpoints for mock UI
└── whatsapp-simulator.module.ts # Module configuration

WhatsappMockUI/
├── index.html                   # Single chat interface
├── style.css                    # WhatsApp-like styling
├── script.js                    # Webhook payload generation + polling
└── README.md                    # Setup and usage instructions
```
