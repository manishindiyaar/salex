# Story 3.5: Booking System API Integration

## Epic
Frontend-Backend Integration - Booking Management

## Story
**As a** business owner  
**I want** to view and manage real customer bookings through the app  
**So that** I can track appointments and revenue accurately

## Acceptance Criteria

### Booking Display Integration
- [ ] Replace mock booking data with real API calls
- [ ] Integrate with `GET /api/v1/businesses/:id/bookings` endpoint
- [ ] Display real booking information in BookingsScreen
- [ ] Show booking details (customer, service, time, status)
- [ ] Handle different booking statuses (pending, confirmed, completed, cancelled)
- [ ] Implement booking filtering and sorting

### Booking Management Operations
- [ ] Integrate booking status updates with backend API
- [ ] Implement booking confirmation functionality
- [ ] Handle booking cancellation from business side
- [ ] Mark bookings as completed
- [ ] Update booking notes and details
- [ ] Real-time booking status updates

### Customer Information Display
- [ ] Show real customer phone numbers in bookings
- [ ] Display customer names when available
- [ ] Handle anonymous customer bookings
- [ ] Link bookings to customer history

### Time Slot Integration
- [ ] Load real available time slots from backend
- [ ] Integrate with `GET /:businessId/timeslots` endpoints
- [ ] Display accurate availability based on existing bookings
- [ ] Handle business hours and service duration calculations
