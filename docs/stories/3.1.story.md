## Test Scenarios
1. **Happy Path**: User enters phone → receives OTP → verifies → authenticated
2. **Session Persistence**: User closes app → reopens → remains authenticated
3. **Token Expiration**: Token expires → user redirected to auth → re-authentication works
4. **Network Issues**: No network → graceful error → retry when connected
5. **Invalid OTP**: Wrong OTP → clear error message → allows retry
6. **Rate Limiting**: Too many requests → appropriate error → cooldown period
