# Story 1.1: Project Scaffolding & Setup

## Status
Draft

## Story
**As a** developer, **I want** the monorepo structure with all required configurations and dependencies installed, **so that** I have a clean and consistent foundation to start building features.

## Acceptance Criteria
1. The `salon-pro` monorepo is created and initialized with Git.
2. Turborepo is configured at the root.
3. The `apps/api` (NestJS) and `apps/merchant-app` (React Native) workspaces are created.
4. The `packages/shared-types`, `packages/eslint-config-custom`, and `packages/typescript-config` workspaces are created.
5. Running `pnpm install` at the root successfully installs all dependencies for all workspaces.
6. A basic "hello world" or default screen runs for both the API and the merchant app.

### Project Structure [Source: architecture/10-unified-project-structure.md]
The monorepo structure must follow this exact layout:
```
salex/
├── apps/
│   ├── api/                 # NestJS backend application
│   │   ├── src/
│   │   │   ├── modules/     # Feature modules (auth, businesses, bookings, notifications, whatsapp)
│   │   │   ├── core/        # Core services (PrismaService, Logger)
│   │   │   └── main.ts      # Main application entry point
│   │   ├── package.json
│   │   └── tsconfig.json
│   └── merchant-app/        # React Native application
│       ├── src/
│       │   ├── components/  # Reusable UI components
│       │   ├── screens/     # Top-level screen components
│       │   ├── navigation/  # React Navigation setup
│       │   ├── services/    # API client services
│       │   ├── store/       # Zustand state management stores
│       │   └── assets/      # Fonts, images, and custom sound file
│       └── package.json
├── packages/
│   ├── shared-types/        # CRITICAL: Shared data model interfaces
│   ├── eslint-config-custom/ # Shared ESLint configuration
│   └── typescript-config/   # Shared tsconfig.json
├── docs/                    # All BMad-generated documents
├── package.json             # Root package.json with workspaces config
└── turbo.json               # Turborepo configuration
```

### Technology Stack [Source: architecture/03-tech-stack.md]
**Core Technologies:**
- **Monorepo Tool**: Turborepo 2.0.x - Manages build caching and task orchestration
- **Package Manager**: pnpm - Required for workspace management
- **Frontend Framework**: React Native 0.74.x - Cross-platform mobile development
- **Backend Framework**: NestJS 10.3.x - TypeScript-first Node.js framework
- **Language**: TypeScript 5.5.x - Used consistently across all workspaces
- **UI Components**: React Native Paper 5.12.x - Material Design components
- **State Management**: Zustand 4.5.x - Simple global state management
- **Testing**: Jest 29.7.x - Unit and integration testing framework

### Development Workflow [Source: architecture/13-development-workflow.md]
**Prerequisites for developers:**
- Node.js (v20.11.0 or later)
- pnpm (package manager)
- Git
- Docker (for future database setup)

**Development Commands:**
- `pnpm dev` - Start all applications
- `pnpm dev:api` - Start only backend API
- `pnpm dev:app` - Start only React Native app
- `pnpm test` - Run all tests
- `pnpm lint` - Lint all code

### Backend Architecture [Source: architecture/12-backend-architecture.md]
**Module Structure:**
The backend uses NestJS modules organized by feature domain:
- `auth/` - Clerk JWT validation and guards
- `businesses/` - Business and services management logic
- `bookings/` - Booking creation and management logic
- `notifications/` - FCM and SMS notifications logic
- `whatsapp/` - WhatsApp webhook and session logic

**Service Pattern:**
Each module contains a controller (HTTP requests) and service (business logic).

### Coding Standards [Source: architecture/17-coding-standards.md]
**Critical Rules:**
- All shared data types must be defined in `packages/shared-types`
- Controllers forbidden from direct Prisma client interaction
- API calls must use singleton apiClient instance
- Zustand state treated as immutable

**Naming Conventions:**
- Components: PascalCase (`BookingListItem.tsx`)
- Service Files: PascalCase (`BookingService.ts`)
- Test Files: `*.test.ts` or `*.spec.ts`

### Testing
**Testing Strategy [Source: architecture/16-testing-strategy.md]:**
- **Frontend Tests**: Jest & React Native Testing Library
  - Unit tests for components alongside source (`PrimaryButton.test.tsx`)
  - Integration tests for complete screens
- **Backend Tests**: Jest & Supertest
  - Unit tests for services (dependencies mocked)
  - Integration tests for controllers (full HTTP flow)
- Test files must be co-located with source code where applicable

**Testing Requirements for this story:**
- Create basic test structure in both workspaces
- Verify Jest configuration works in all packages
- No complex test cases needed - just verify testing framework setup

### File Locations
**Exact paths for new code:**
- Root workspace config: `salex/package.json`, `salex/turbo.json`
- API workspace: `salex/apps/api/package.json`, `salex/apps/api/src/main.ts`
- Merchant app workspace: `salex/apps/merchant-app/package.json`, `salex/apps/merchant-app/src/`
- Shared packages: `salex/packages/*/package.json` and respective `src/index.ts` files

### Technical Constraints
- Must use pnpm for package management (not npm or yarn)
- TypeScript 5.5.x required across all workspaces
- Turborepo configuration must enable build caching
- All workspaces must follow the established naming conventions
