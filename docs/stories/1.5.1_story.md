# Story 1.5.1: Complete Frontend Onboarding UI Implementation

**Type:** Frontend-only Implementation
**Priority:** High
**Status:** Ready for Development
**Linked to:** [Epic 1 - Foundation and Merchant Onboarding](../prd/6-epic-1-foundation-and-merchant-onboarding.md)
**Based on:** [Frontend UX Specification](../architecture/11-1-frontend-ux-specification.md)

## Story Brief
As a new salon owner, I want a complete, guided onboarding experience that helps me set up my business profile and services from start to finish, with a modern, responsive interface that works seamlessly across all devices.

## Acceptance Criteria

### 1. Phone Authentication Flow
- [ ] **Phone Number Input Screen**
  - International phone number input with country code dropdown
  - **Phone number field**: Format validation (XXX-XXX-XXXX)
  - Send OTP with error handling
  - Loading state while requesting OTP
  - Error messages for invalid formats
  - Visual feedback with phone icon animation

- [ ] **OTP Verification Screen**
  - 4-digit OTP code input (6 boxes with auto-advance)
  - Resend OTP functionality (60-second cooldown)
  - Error handling for invalid/expired OTPs
  - Loading state during verification
  - "Verify phone number" confirmation message
  - Auto-advance between input fields

### 2. Business Type Selection
- [ ] **Visual Type Selection Screen**
  - Card-based selection with hover animations
  - Business types: **Salon, Spa, Barber Shop, Nail Studio, Wellness Center**
  - Each type has distinct visual cards with:
    - Icon (animated on hover)
    - Type name and subtitle
    - Micro-description
    - **Selection feedback** (border color change, tick animation)

### 3. Business Profile Onboarding Wizard (5-Step Process)

#### Step 1: Business Information
- **Business Name** [Required]
  - Minimum 3 characters, auto-focus
  - Real-time name validation
- **Owner Name** [Required] 
  - Full name input
- **Email Address** [Required]
  - Format validation with visual indicators
- **Business Phone** [Optional]
  - Same format as authentication phone
- **Website URL** [Optional]
  - URL format validation

#### Step 2: Business Address (Google Places Integration)
- **Location Input**
  - Google Places autocomplete for address
  - Map visualization with pin drop animation
  - Manual address entry toggle
  - **Required Fields**: Street Address, City, State, ZIP

#### Step 3: Business Hours Configuration
- **Weekly Schedule Grid**
  - Checkboxes for each day
  - Time pickers (09:00 AM - 05:00 PM)
  - "Closed" day toggle
  - **Quick setup**: "Copy to other days" functionality
  - Real-time preview

#### Step 4: Service Setup
- **Service Creation Interface**
  - Add multiple services with dynamic form
  - **Required per service**:
    - Service Name (50 char limit)
    - Duration (15min - 4hrs)
    - Price ($0 - $1000)
    - Description (optional, 200 char limit)
  - Service card preview with images
  - Drag to reorder capability
  - **Validation**: At least one service required

#### Step 5: Review & Confirmation
- **Complete Profile Preview**
  - All information on summary cards
  - **Edit buttons** for each section
  - Terms of Service checkbox
  - "Launch Business" button
  - **Success Animation** (confetti/fade-in)

### 4. React Navigation Integration
- [ ] **Navigation Configuration** using React Navigation 6.x
  - Nested navigators for auth vs onboarding flows
  - Shared header with progress indicator
  - Swipe gestures disabled during onboarding
  - **Back navigation** handling with prompts
  - **Deep linking** support

### 5. Form Validation & Error Handling
- [ ] **Client-side validation**
  - Real-time field validation
  - Error states with red borders
  - Help text and error messages
  - **Focus management** on validation errors
  - Form submission prevention until valid

### 6. Loading States & Micro-interactions
- [ ] **Skeleton loading** for async operations
- [ ] **Button animations** on press
- [ ] **Progress indicators** for multi-step forms
- [ ] **Toast notifications** for success/error states
- [ ] **Fade/slide transitions** between screens

### 7. Responsive Design Requirements
- [ ] **Mobile-first approach** (iOS/Android)
- [ ] **Tablet optimization** (landscape/portrait)
- [ ] **Accessibility compliance** (WCAG 2.1)
  - Screen reader labels
  - Keyboard navigation support
  - Focus indicators
  - Color contrast ratios

## Technical Implementation Details

### Screen Navigation Tree
```
RootNavigator
├── AuthNavigator
│   ├── PhoneInputScreen
│   └── OTPVerificationScreen
└── OnboardingNavigator
    ├── BusinessTypeScreen
    └── BusinessProfileWizard
        ├── Step1_BusinessInfo
        ├── Step2_Address
        ├── Step3_Hours
        ├── Step4_Services
        └── Step5_Review
```

### Component Architecture
- **Reusable Components**: 
  - FormInput (with validation)
  - BusinessTypeCard
  - ServiceCard
  - ProgressBar
  - LoadingSpinner

### State Management
- **Context**: OnboardingContext
- **Local state**: Form data, validation errors
- **Async storage**: Partial form save
- **Navigation param** sharing

### API Integration
- **Clerk Auth**: Custom hooks for phone auth
- **Google Places**: Address autocomplete
- **Backend API**: Business creation, service management
- **Image Upload**: Business logo (optional for future)

## Visual Design System
- **Theme**: Light mode only for v1
- **Typography**: System defaults with fallbacks
- **Colors**: Primary blue (#007AFF), secondary gray
- **Animations**: Spring animations with easing
- **Icons**: SF Symbols (iOS) / Material Icons (Android)
