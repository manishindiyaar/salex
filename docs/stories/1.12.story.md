# Story 1.12: Complete Merchant Onboarding Backend Flow

**Epic**: Foundation & Merchant Onboarding  
**Story ID**: 1.12  
**Priority**: Critical (P1 - New User Acquisition)  
**Estimated Effort**: 13 story points  
**Status**: Ready for Development
**Focus**: Do not focus on Backend API Support Just work for Frontend Onboarding UI


Use JSON for feeding the data and functionalities like api.


## Story

**As a** new salon owner downloading the Salex merchant app,  
**I want** to complete a guided onboarding flow from phone verification to fully functional business,  
**so that** I can start accepting bookings within 5 minutes of first app launch.

## User Journey Map

### Key Moments That Matter
1. **First Impression** (0-15s): App launches, clear value proposition
2. **Frictionless Entry** (15s-2min): Phone number → OTP → Business type selection  
3. **Guided Setup** (2-5min): Multi-step wizard collects business info progressively
4. **Instant Gratification** (5min+): Live QR code generation + first booking capability
5. **Retention Hook** (Day 0-7): Success tracking and continuous profile optimization

## Acceptance Criteria

### AC1: Phone Authentication Flow
**Given** I just installed the Salex app  
**When** I open the app for the first time  
**Then** I see:
- Welcome screen with clear value proposition: "Turn your salon into a booking machine"
- Phone number input with country code selector (default: +91 India)
- Input validation for 10-digit phone numbers
- "Send OTP" button becomes active only after valid phone number
- Loading state with spinner during OTP delivery
- Clear error states for invalid numbers or network failures

### AC2: OTP Verification Screen
**Given** I've submitted my phone number  
**When** I receive the OTP  
**Then** I can:
- Enter 6-digit OTP in auto-focused input field
- See countdown timer (3 minutes) with resend capability
- Resend OTP instantly if timer expires (1-minute backoff)
- View clear error messages for invalid/expired OTPs
- Navigate back to phone screen to edit number if needed
- Skip to business type selection (for development/testing only with toggle in debug menu)

### AC3: Business Type Selection
**Given** I've successfully verified my phone  
**When** I'm redirected to business type selection  
**Then** I see:
- Bold headline: "Tell us about your business"
- Visual cards with icons for business types:
  - 💄 **Salon** (Hair, beauty, makeup)
  - 💅 **Spa** (Wellness, therapy, relaxation)  
  - 💇 **Barber Shop** (Men's grooming)
  - 🧴 **Clinic** (Skincare, aesthetics)
  - **Custom Business** (For other service types)
- Single-select interaction with subtle animation
- Continue button becomes active only after selection
- Skip option (creates "SALON" type by default)

### AC4: Multi-Step Business Profile Setup
**Given** I've selected my business type  
**When** the onboarding wizard starts  
**Then** the flow progresses through these screens:

**Step 1: Business Identity**
- Business name input with placeholder examples
- Tagline/description field (optional, 80 char limit)
- Upload business logo (photo library or camera)
- Preview of business profile picture

**Step 2: Contact & Location**  
- Primary contact number (pre-filled from auth)
- WhatsApp business number (optional)
- Address with location auto-complete (Google Places API)
- Map preview of business location

**Step 3: Services & Pricing**
- "Add your first service" guided creation
- Service name input with suggestions
- Price input with currency auto-formatting
- Duration selector (15min, 30min, 45min, 60min, 90min, 2hr, 3hr, custom)
- "Add another service" capability (unlimited)
- Skip option (creates sample "Haircut" service)

**Step 4: Business Hours Setup**
- Visual weekly calendar view
- Individual day toggle (open/closed)
- Time range inputs for each open day
- "Copy to all weekdays" for Monday settings
- Special scheduling capabilities
- Instant preview of booking availability

**Step 5: Review & Complete**
- Business profile preview card
- All services listed with pricing
- Business hours overview
- Estimated daily earning potential
- QR code preview (ready to print)
- "Go Live" button activation

### AC5: Data Persistence & Sync
**Given** I'm progressing through the onboarding  
**When** I complete each section  
**Then**:
- All data auto-saves to backend after each step
- Offline capability stores locally in case of network issues
- Resume capability shows progress from last completed step
- Automatic retry for failed API calls
- Conflict resolution when network returns

### AC6: Loading & Error States
**Given** API calls are in progress  
**When** users expect immediate feedback  
**Then**:
- Smooth animated loading indicators between steps
- Graceful degradation when services are unavailable  
- Offline mode with cached data for critical flows
- Retry logic with exponential backoff for transient failures
- Network error boundaries with actionable next steps

### AC7: Analytics & Tracking
**Given** Users complete onboarding  
**When** The system tracks their journey  
**Then**:
- Time to complete each onboarding step
- Drop-off points in the funnel
- Most common business types selected
- Service categories most frequently added
- Device platform and OS version tracking

## Technical Requirements

### Screens Structure
```
src/screens/
├── auth/
│   ├── PhoneAuthScreen.tsx
│   └── OtpVerificationScreen.tsx
├── onboarding/
│   ├── BusinessTypeScreen.tsx
│   ├── BusinessIdentityScreen.tsx (Step 1)
│   ├── ContactLocationScreen.tsx (Step 2)  
│   ├── ServicesPricingScreen.tsx (Step 3)
│   ├── BusinessHoursScreen.tsx (Step 4)
│   ├── ReviewCompleteScreen.tsx (Step 5)
│   └── OnboardingNavigator.tsx
```

### Component Library Usage
- **React Native Paper**: Buttons, Cards, TextInputs, AppBar
- **React Native Vector Icons**: Feather icons for business types
- **React Native Image Picker**: Logo and photo uploads
- **React Native Maps**: Location picker and business address setup

### Navigation Flow
```plaintext
PhoneAuthScreen → OtpVerificationScreen → BusinessTypeScreen → MultiStepWizard → Dashboard
                                      ↓
                                  AuthStack (if logged out)
```

### State Management
- **Zustand store**: `onboardingStore.ts` for wizard state
- **AsyncStorage**: Persistent storage for incomplete onboarding
- **React Query**: Server state management for business APIs

### API Integration Points (but make it optional i initially need UI)
- `POST /api/v1/auth/phone` - Phone authentication start
- `POST /api/v1/auth/verify` - OTP verification  
- `POST /api/v1/businesses` - Create initial business
- `POST /api/v1/businesses/:id/services` - Add services individually
- `PUT /api/v1/businesses/:id/hours` - Set business hours
- `GET /api/v1/businesses/:id` - Get business for preview

### Validation Rules
- **Phone**: 10 digits, Indian numbers starting with 6-9
- **Business Name**: 2-50 characters, no special symbols except &,-
- **Service Name**: 3-40 characters, descriptive service names
- **Price**: 0.01-9999.99, positive decimal values  
- **Duration**: 15-480 minutes, in 15-minute increments
- **Address**: At least 10 characters, autocomplete required

### User Experience Details

#### Micro-interactions
1. **Phone input** - Animated underline focus
2. **OTP fields** - Auto-advance between digit inputs  
3. **Business cards** - Scale animation on selection
4. **Progress** - Smooth linear progression across 5 steps
5. **Form validation** - Real-time inline error messages
6. **Success states** - Lottie animation for completed onboarding

#### Loading States
- **Phone verification** - 2-3 seconds with progress ring
- **OTP verification** - 1-2 seconds with checkmark animation  
- **Business creation** - 3-5 seconds with "Setting up your business..."
- **Service addition** - Individual service creation with realtime feedback
- **Image upload** - Preview thumbnail with upload progress

#### Error Handling
- **Network errors** - "Connection issues" with retry button
- **Server errors** - "Something went wrong" fallback messages  
- **Validation errors** - Inline field validation with helpful error text
- **Upload failures** - Error toast with re-upload button

### Performance Considerations
- **Image compression** - Compress images > 2MB before upload
- **Lazy loading** - Load screens on-demand, not upfront
- **Code splitting** - Separate auth bundle from main app
- **Offline capability** - Cache business types and user selections
- **Memory management** - Clear image cache after upload completion

### Testing Strategy
- **Unit tests** - Component rendering and state management
- **Integration tests** - API calls and user flow validation  
- **E2E tests** - Complete onboarding flows with real API
- **Accessibility** - VoiceOver/TalkBack screen reader support
- **Cross-platform** - iOS and Android device testing

### Security Requirements
- **Phone verification** - Rate limiting for SMS resend
- **Image upload** - Virus scanning and file type validation
- **API calls** - JWT token injection via auth interceptor
- **Data encryption** - Secure storage for offline form data
- **Input sanitization** - XSS protection for all user inputs
