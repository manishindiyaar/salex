# Story 3.3: Business Onboarding API Integration

## Epic
Frontend-Backend Integration - Business Management

## Story
**As a** business owner  
**I want** my business onboarding data to be saved to the backend  
**So that** my business information persists and I can manage it from any device

## Acceptance Criteria

### Business Creation Integration
- [ ] Replace mock business creation with real API calls
- [ ] Integrate with `POST /api/v1/business` endpoint
- [ ] Send all onboarding data (name, type, phone, address, hours)
- [ ] Handle successful business creation response
- [ ] Store returned business ID and data locally
- [ ] Navigate to main app after successful creation

### Business Profile Management
- [ ] Integrate with `GET /api/v1/business/me` endpoint
- [ ] Display real business data in profile screens
- [ ] Update business information via `PUT /api/v1/business/:id`
- [ ] Handle business hours updates with API integration
- [ ] Sync local state with backend data

### Onboarding Flow Updates
- [ ] Replace mock data with API-driven onboarding
- [ ] Maintain existing UI/UX without breaking changes
- [ ] Add loading states for API operations
- [ ] Handle network errors gracefully during onboarding
- [ ] Implement retry mechanism for failed requests

### Data Persistence
- [ ] Use real business data throughout the app
- [ ] Remove dependency on mock business data
- [ ] Ensure data consistency between screens
- [ ] Handle offline scenarios appropriately

## Technical Requirements

### API Integration Points
```typescript
// Business Service Methods
createBusiness(data: CreateBusinessRequest): Promise<Business>
getMyBusiness(): Promise<Business>
updateBusiness(id: string, data: UpdateBusinessRequest): Promise<Business>
updateBusinessHours(id: string, hours: BusinessHours): Promise<Business>
```

### Data Flow
- Onboarding store → API call → Backend → Response → Local storage
- Profile screens load from API rather than mock data
- Real-time updates reflect in UI immediately

### Error Handling
- Network connectivity issues
- Validation errors from backend
- Duplicate business creation attempts
- Authentication token issues

## Definition of Done
- [ ] All business onboarding flows use real API
- [ ] Business profile displays real backend data
- [ ] Business updates persist to backend successfully
- [ ] Error handling works for all edge cases
- [ ] Loading states provide good user experience
- [ ] No mock business data remains in the codebase
- [ ] All existing onboarding tests pass

## Test Scenarios
1. **Complete Onboarding**: New user completes all onboarding steps → business created in backend
2. **Profile Loading**: Existing user opens app → real business data loaded from backend
3. **Business Updates**: User updates business info → changes saved to backend
4. **Network Errors**: No connection during onboarding → graceful error handling
5. **Validation Errors**: Invalid business data → appropriate error messages
6. **Hours Management**: User sets business hours → hours saved and loaded correctly

## Implementation Notes
- Focus on service layer integration first
- Keep existing UI components unchanged initially
- Use shared-types for all request/response objects
- Implement proper loading states for better UX
- Consider caching business data for offline access
