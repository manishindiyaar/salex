WhatsApp Cloud API Masterclass 🟢

Welcome to the definitive guide for integrating WhatsApp into Salex. This document is designed for developers who are new to the WhatsApp Cloud API. We will cover everything from the basic "Hello World" to complex interactive booking flows.

Why Cloud API? Unlike the old "On-Premise" API, the Cloud API is hosted by Meta. It's free to start, scales automatically, and supports the latest interactive features like List Messages and Reply Buttons which are crucial for our booking system.

1. Setup & Configuration

Before writing code, you need to configure the Meta Developer Portal.

  1. Create App: Go to developers.facebook.com and create a "Business" app.
  2. Add WhatsApp: Add the "WhatsApp" product to your app.
  3. Get Token: You'll get a temporary access token. For production, you need a System User token.
  4. Phone Number: Meta gives you a test number. You must verify your own number to go live.

Environment Variables

Add these to your `.env` file:

WHATSAPP_API_URL=https://graph.facebook.com/v18.0
WHATSAPP_ACCESS_TOKEN=EAAG...
WHATSAPP_PHONE_NUMBER_ID=105...
WHATSAPP_WEBHOOK_VERIFY_TOKEN=my_secret_token

2. Understanding Webhooks

WhatsApp doesn't "poll". It "pushes". When a user sends a message, Meta hits your server endpoint (POST /api/v1/webhooks/whatsapp).

Verification (The Handshake)

When you first add the webhook URL in the dashboard, Meta sends a GET request to verify you own the server.

// NestJS Controller Example
@Get()
verifyWebhook(@Query() query: any) {
  const mode = query['hub.mode'];
  const token = query['hub.verify_token'];
  const challenge = query['hub.challenge'];

  if (mode === 'subscribe' && token === process.env.WHATSAPP_WEBHOOK_VERIFY_TOKEN) {
    return challenge; // You MUST return this exact challenge string
  }
  throw new ForbiddenException();
}

The Payload Structure

The incoming POST payload is deeply nested. Here is the simplified structure you need to care about:

  "object": "whatsapp_business_account",

3. Interactive Payloads (The Fun Stuff)

Typing is hard. Buttons are easy. Salex relies heavily on Interactive Messages to guide users.

Sending a List Message (Service Selection)

Use this when you have 2-10 options (e.g., list of services).

// Outgoing Payload to Graph API
{
  "messaging_product": "whatsapp",
  "to": "919876543210",
  "type": "interactive",
  "interactive": {
    "type": "list",
    "header": { "type": "text", "text": "Our Services" },
    "body": { "text": "Please select a service to book:" },
    "footer": { "text": "Salex Booking" },
    "action": {
      "button": "View Menu",
      "sections": [
        {
          "title": "Hair",
          "rows": [
            { "id": "svc_001", "title": "Haircut", "description": "$20 - 30 mins" },
            { "id": "svc_002", "title": "Coloring", "description": "$50 - 60 mins" }
          ]
        }
      ]
    }
  }
}

Receiving the Selection

When the user clicks "Haircut", you get this webhook:

{
  "type": "interactive",
  "interactive": {
    "type": "list_reply",
    "list_reply": {
      "id": "svc_001", // <--- THIS IS WHAT YOU NEED
      "title": "Haircut",
      "description": "$20 - 30 mins"
    }
  }
}

Sending Reply Buttons (Time Slots)

Use this for up to 3 options (e.g., "Confirm", "Cancel", "Change Time").

{
  "type": "interactive",
  "interactive": {
    "type": "button",
    "body": { "text": "Confirm booking for Haircut at 10:00 AM?" },
    "action": {
      "buttons": [
        { "type": "reply", "reply": { "id": "confirm_booking", "title": "✅ Confirm" } },
        { "type": "reply", "reply": { "id": "cancel_booking", "title": "❌ Cancel" } }
      ]
    }
  }
}

6. Code Implementation Strategy

Do not write a giant if/else block. Use a router pattern.

// whatsapp.service.ts

async processMessage(message: WhatsAppMessage) {
  // 1. Get User Session
  let session = await this.sessionService.get(message.from);
  
  // 2. Route based on State
  switch (session.state) {
    case 'GREETING':
      return this.handleGreeting(session, message);
    case 'BUSINESS_SELECTED':
      return this.handleServiceSelection(session, message);
    case 'SLOT_SELECTION':
      return this.handleSlotSelection(session, message);
    case 'CONFIRMATION':
      return this.handleConfirmation(session, message);
    default:
      return this.resetFlow(session);
  }
}

async handleServiceSelection(session, message) {
  // Check if it's a list reply
  if (message.type !== 'interactive' || message.interactive.type !== 'list_reply') {
    return this.sendText(message.from, "Please select a service from the list.");
  }

  const serviceId = message.interactive.list_reply.id;
  // Save selection to session
  await this.sessionService.update(message.from, { 
    state: 'SLOT_SELECTION', 
    tempData: { serviceId } 
  });

  // Fetch slots and send next list
  const slots = await this.slotService.getSlots(session.businessId);
  return this.sendSlotList(message.from, slots);
}
